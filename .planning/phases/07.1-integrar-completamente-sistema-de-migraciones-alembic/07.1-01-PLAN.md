# Plan: Phase 07.1 — Integrar completamente sistema de migraciones Alembic

## Context

Alembic 1.12.1 is already installed as a transitive dependency but is not configured for this project. The project currently uses `Base.metadata.create_all()` in `models/database.py::init_db()` to create all 32 tables on every startup. A standalone one-off migration script `migrations/add_invite_link_to_channels.py` has already been executed (the `invite_link` column exists in the `channels` table). Phase 07.1 replaces all of this with a proper Alembic migration system.

## Deliverables

1. `alembic.ini` and `alembic/` directory configured with `env.py` pointing to `models.database.Base`
2. Initial baseline migration capturing all 32 existing tables
3. Both SQLite dev DB and Railway PostgreSQL DB stamped to the baseline
4. `alembic==1.12.1` added to `requirements.txt`
5. `init_db()` in `models/database.py` updated to be a no-op (Alembic owns schema)
6. Standalone migration `migrations/add_invite_link_to_channels.py` archived to `migrations/archive/`
7. `railway.toml` updated to run `alembic upgrade head` before `python bot.py`
8. `migrations/README.txt` created pointing to `alembic/`

## must_haves (goal-backward verification)

- `alembic --version` outputs `alembic, version 1.12.1`
- `alembic.ini` exists and contains `prepend_sys_path = .`
- `alembic/env.py` exists and imports `from models.database import Base` and `from config.settings import bot_config`
- `alembic/env.py` calls `config.set_main_option("sqlalchemy.url", bot_config.DATABASE_URL)`
- `alembic/script.py.mako` exists
- `alembic/versions/` directory exists
- `requirements.txt` contains `alembic==1.12.1`
- `migrations/README.txt` exists and contains the string `alembic`
- `migrations/archive/` directory exists and contains `add_invite_link_to_channels.py`
- `railway.toml` contains `alembic upgrade head`
- `init_db()` in `models/database.py` contains a comment referencing Alembic

- Research section "Pattern 2: Baseline Migration via Autogenerate"
</read_first>

<action>
From `/data/data/com.termux/files/home/repos/lucien_bot/`, run:

```
alembic revision --autogenerate -m "initial schema baseline"
```

This reads `models.database.Base` metadata and compares it to the live SQLite database `lucien_bot.db`. Since all 32 tables already exist and the schema is in sync with models, the generated revision will be a near-empty file with a correct upgrade() that does nothing when run on an already-migrated DB, but correctly creates all tables on a fresh DB.

If the command prompts about a "None SQLAlchemy url", ensure `dotenv` is loading `.env` before running, or set the variable inline:

```
DATABASE_URL="sqlite:///lucien_bot.db" alembic revision --autogenerate -m "initial schema baseline"
```

The output file will be created at `alembic/versions/<revision_id>_initial_schema_baseline.py`.
</action>
