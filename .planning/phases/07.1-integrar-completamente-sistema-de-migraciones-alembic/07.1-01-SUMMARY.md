## Deviation from Plan

### 2 revisions instead of 1
The autogenerate baseline (`c25b0020dcc5`) detected 3 schema differences between the Python models and the live SQLite DB:
- `story_achievements.icon` column missing from DB
- `story_choices.choice_archetype` column missing from DB
- `promotion_interests` unique constraint missing from DB

A second migration (`53f8f3496f23`) was generated and applied to sync the DB.

### SQLite unique constraint limitation
`alembic check` reports a model/DB mismatch on `promotion_interests` because:
- The Python model defines `UniqueConstraint('user_id', 'promotion_id', name='uq_user_promotion_interest')`
- SQLite does not support `ALTER TABLE ADD CONSTRAINT`, so the migration uses `create_index(..., unique=True)` instead
- This is functionally equivalent and will work correctly on PostgreSQL (Railway)
