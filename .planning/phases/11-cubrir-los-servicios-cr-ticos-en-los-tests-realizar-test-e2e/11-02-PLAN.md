**Race condition to fix:** In `complete_order`, the product stock decrement at `product.stock -= order_item.quantity` is not protected by locking. Two concurrent calls can both read `stock=1`, both pass, and both decrement. Fix: add `with_for_update()` to the product query in `complete_order`, or use `get_product` after locking. The simplest fix is changing:
```python
product = order_item.product
```
to query the product with `with_for_update()` inside the same session.

**Test pattern to follow:** Existing tests use `db_session` fixture and mock chain verification for `with_for_update()` calls (see test_vip_service.py TestVIPServiceRaceCondition).
</context>
