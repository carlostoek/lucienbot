# Plan 14-04: Deployment y Configuración

**Goal:** Configurar URLs de WebApp para desarrollo y producción

**Affected Components:**
- `config/settings.py` — Agregar WEBAPP_URL
- `railway.toml` / `Dockerfile` — Servir archivos estáticos
- `bot.py` — Configurar modo WebApp si es necesario

---

## Task 1: Configurar variables de entorno

Update configuration to support WebApp URLs:

**In `config/settings.py`:**
```python
# WebApp Configuration
WEBAPP_URL = os.getenv(
    "WEBAPP_URL",
    "https://tudominio.com/webapp/dice.html"  # Production default
)
WEBAPP_DEV_URL = os.getenv(
    "WEBAPP_DEV_URL",
    "http://localhost:8080/webapp/dice.html"  # Development
)
```

**For local development:**
Add to `.env` file (don't commit):
```
WEBAPP_URL=http://localhost:8080/webapp/dice.html
```

**Files to modify:**
- `config/settings.py`

---

## Task 2: Servir archivos estáticos en desarrollo

Add static file serving for development:

**Options:**

1. **Using built-in http.server (simplest for dev):**
```bash
# Add to startup script or Makefile
python -m http.server 8080 --directory webapp/
```

2. **Using FastAPI/Flask route in bot (recommended):**
```python
# In webapp_router.py or bot.py
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles

app = FastAPI()
app.mount("/webapp", StaticFiles(directory="webapp"), name="webapp")
```

3. **Using aiogram's built-in:**
If using webhook mode, can serve statics alongside.

**Files to create/modify:**
- `routers/webapp_router.py` (if using FastAPI approach)
- `bot.py` (if integrating directly)

---

## Task 3: Configuración Railway/Nginx para producción

Configure production static file serving:

**Option 1: Railway + Nginx (if using nginx buildpack):**
```nginx
# nginx.conf
location /webapp {
    alias /app/webapp;
    try_files $uri $uri/ =404;
}
```

**Option 2: Railway + Python (serve via FastAPI):**
Mount static files as in Task 2.

**Option 3: Separate static hosting:**
- Host on GitHub Pages
- Host on Cloudflare Pages
- Host on any CDN

Update `railway.toml` if needed for static file serving.

**Files to modify:**
- `railway.toml`
- `Dockerfile` (if exists)
- `nginx.conf` (if using nginx)

## Task 4: Configurar WebApp en BotFather

Document or configure the WebApp URL in BotFather:

**Steps:**
1. Go to @BotFather
2. Select your bot
3. Go to "Bot Settings" → "Menu Button" → "Configure menu button"
4. Or use `/setmenubutton` command
5. Set button text: "🎲 Lanzar dados"
6. Set WebApp URL: Your production URL

**For testing in development:**
- Use ngrok to expose localhost: `ngrok http 8080`
- Temporary URL for testing with BotFather

**Files to create:**
- Document in `docs/webapp-setup.md` or similar
